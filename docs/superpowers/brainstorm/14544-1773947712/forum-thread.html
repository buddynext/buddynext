Best practices for TypeScript generics? Struggling with conditional types
AJ
Alex Johnson Moderator
· posted 45 minutes ago
I've been working on a utility type library and keep running into issues with conditional types. Specifically, distributive conditional types are tripping me up. Here's a simplified example of what I'm trying to do:
type IsArray<T> = T extends any[] ? true : false; // This distributes over unions, which I don't want type Test = IsArray<string | string[]>; // boolean, not false | true
Has anyone found a clean pattern for preventing distribution?

Great question! The key is understanding distributive vs non-distributive conditionals. Wrap your type parameter in a tuple to prevent distribution:
type IsArray<T> = [T] extends [any[]] ? true : false;
The [T] wrapper prevents TypeScript from distributing the conditional over union members. Now IsArray<string | string[]> correctly returns false (string isn't an array type as a whole).
type Test = IsArray<string | string[]>; // false ✓

One more thing — if you're doing a lot of conditional types, the infer keyword becomes your best friend. Combined with the tuple pattern Sarah showed, you can build some really powerful utilities.
type UnwrapArray<T> = [T] extends [Array<infer U>] ? U : T;
