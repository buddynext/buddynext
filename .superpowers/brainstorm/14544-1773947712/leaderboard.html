📋 Leaderboard — Spec Notes
Routing & Integration
• URL: /community/leaderboard/
• Template: templates/views/leaderboard.php
• Integration: WBGamification bridge — BuddyNext reads from WBGamification's points + badges tables
• Standalone fallback: if WBGamification not active, page renders "Install WBGamification to enable leaderboard" notice
• REST: GET buddynext/v1/leaderboard?period=month&category=contributors
Data & Points Logic
• Points earned via bn_action hooks: post.created, comment.added, reaction.given, space.contributed, follow.made
• Points configurable per action in WBGamification → Settings → Point Rules
• Period reset: weekly (Monday 00:00 UTC), monthly (1st 00:00 UTC), all-time cumulative
• Category "Top Connectors" ranks by follow/friend count; "Rising Stars" ranks by % growth last 7 days
• Badges awarded automatically by WBGamification on threshold events
Frontend Architecture
• PHP template + WP Interactivity API — no build step
• Period & category tabs swap data via wp_interactivity navigate() — no page reload
• Podium renders top 3; table renders ranks 4–50 (paginated in 10s)
• "Your Rank" row always visible; uses current user's rank from REST response
• Badge tooltip: CSS :hover + aria-label (no JS tooltip library)
• Dark mode: CSS custom properties, toggleDark() stores pref in localStorage
