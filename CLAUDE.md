## What Is BuddyNext

Enterprise-grade social community platform for WordPress (free + pro). Owned by Wbcom Designs.

- **Site URL (local):** http://forums.local
- **Plugin path:** `wp-content/plugins/buddynext/`
- **Namespace:** `BuddyNext\*` (free) / `BuddyNextPro\*` (pro)
- **REST namespaces:** `buddynext/v1` (free) ¬∑ `buddynext-pro/v1` (pro)
- **Bootstrap hook:** `plugins_loaded:15` ‚Üí `BuddyNext\Core\Plugin::init()` ‚Üí fires `buddynext_loaded`
- **PHP:** 8.1+ ¬∑ **WP:** 6.9+ (Abilities API required)

## Non-Negotiable Standards ‚Äî Read Before Every Task

### 1. Enterprise Code Quality ‚Äî No Shortcuts

- Every file must pass WPCS before committing. Run `mcp__wpcs__wpcs_check_file` on every PHP file you write or modify.
- Every class must pass PHPStan level 5+. Run `mcp__wpcs__wpcs_phpstan_check` after writing new classes.
- No `@todo`, no stub implementations, no `/* TODO */` ‚Äî ship complete code or don't ship.
- **Zero AI markers** ‚Äî no `// Generated by`, no `// AI-assisted`, no `// Claude`, no `// This code was...`, no `@generated`. Code reads as if written by a senior WordPress engineer. No exceptions.
- No `echo` in production paths ‚Äî use `wp_send_json_*`, templates, or REST responses.
- All DB queries use `$wpdb->prepare()`. Zero raw interpolation.
- All nonces validated on every state-changing request.
- Capabilities checked on every admin and REST endpoint.
- Sanitize input at entry. Escape output at exit. Always.

### 2. WPCS MCP ‚Äî Mandatory Workflow

Run after writing **any** PHP file:

```
mcp__wpcs__wpcs_check_file({ file_path: "...", standard: "WordPress" })
```

Run after writing a batch of files:

```
mcp__wpcs__wpcs_check_directory({ directory: "includes/...", standard: "WordPress" })
```

Run before any commit:

```
mcp__wpcs__wpcs_pre_commit({ standard: "WordPress" })
```

Fix all errors and warnings before proceeding. WPCS is not optional ‚Äî it is part of Done.

### 3. Test-Driven Development ‚Äî Mandatory

Write the failing test FIRST. Then write the implementation. Then make it pass.

```
vendor/bin/phpunit tests/[Area]/[ClassTest].php --testdox
```

Never mark a task as complete unless tests pass.

### 4. Premium UX ‚Äî Non-Negotiable

Every template must match the HTML mockups in `.superpowers/brainstorm/14544-1773947712/`. Pixel-level fidelity is expected for:
- Color tokens (use CSS custom properties verbatim from style-guide.html)
- Typography (Inter 15px body, Plus Jakarta Sans display)
- Spacing (4px grid: --s1 through --s12)
- Dark mode (all components must support `[data-theme="dark"]`)
- Mobile (every layout ‚â§640px must be tested ‚Äî full-width, no horizontal scroll)
- Interactions (hover states, focus rings, loading states ‚Äî all from mockups)

## Plugin Architecture

### Bootstrap Chain
```
plugins_loaded (priority 10) ‚Üí WPMediaVerse, Jetonomy (addons)
plugins_loaded (priority 15) ‚Üí BuddyNext\Core\Plugin::init() ‚Üí buddynext_loaded
plugins_loaded (priority 20) ‚Üí BuddyNext Pro hooks
                              ‚Üí Bridge classes (if addons active)
```

### Service Container
Same DI pattern as WPMediaVerse:
```php
// Bind
$container->bind( 'social_graph', fn() => new \BuddyNext\SocialGraph\FollowService() );

// Resolve (singleton)
$container->get( 'social_graph' );

// Global helper
buddynext_service( 'social_graph' );
```

### Permission System
Single entry point for every gate:
```php
buddynext_can( $user_id, 'view-space', [ 'space_id' => $space_id ] )
buddynext_can( $user_id, 'post-in-feed' )
buddynext_can( $user_id, 'send-dm', [ 'recipient_id' => $recipient_id ] )
```

### Abilities Registration
All abilities registered at boot via WordPress Abilities API:
```php
wp_register_ability( 'buddynext-view-space',   [ 'label' => 'View Space' ] );
wp_register_ability( 'buddynext-post-in-feed', [ 'label' => 'Post in Feed' ] );
// etc.
```
